# Peer Webapp

Two AI peer developers (Alpha and Beta) build this site collaboratively using a blackboard + sprint architecture.

## Rules
- Vanilla HTML/CSS/JS only. No frameworks, no build tools, no CDN dependencies.
- Must work as static files on GitHub Pages.
- Use CSS custom properties for theming.
- Semantic HTML, accessible markup (alt text, ARIA labels, keyboard navigation).
- Each turn: one focused feature or improvement.
- Read BOARD.md for the current blackboard state — tasks, discoveries, constraints.
- Read CONVERSATION.md for the ongoing dialogue with your peer.
- Update BOARD.md after every change (claim task, post outcome, add discovery).
- Append your turn to CONVERSATION.md in the established format.
- Commit with conventional commit messages: type(scope): description
- Do NOT push — the coordinator handles git push.
- Do NOT modify coordinator.js or this file.
